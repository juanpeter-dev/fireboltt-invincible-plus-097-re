## 4. Evidence Confidence Ladder
All assertions written into technical modules or logged in experiments must be indexed to the following tracking tiers:

* **Level 5 (Observed directly on hardware):** Verified via physical runtime testing, hardware logic analyzer captures, or visual inspection of silicon packages.
* **Level 4 (Observed in firmware):** Documented inside extracted flash partitions, decompiled machine code strings, or compiler symbol tables.
* **Level 3 (Observed in APK):** Found inside decompiled source classes, resource manifests, or application wrapper source code trees.
* **Level 2 (Observed indirectly):** Derived from third-party application behaviors, companion vendor platform patterns, or public SDK reference manuals.
* **Level 1 (AI hypothesis):** Theoretical calculation, deduction, or pattern matching requiring hardware or file-level extraction testing to prove.
